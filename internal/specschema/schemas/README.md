# Pinned specification JSON Schemas

These schemas are embedded into `apibconv` and used by `apibconv --validate` for OpenAPI and AsyncAPI documents.
They are pinned local copies so validation does not depend on network access at runtime.
License and attribution details are in `THIRD_PARTY_NOTICES.md`.

Sources:

- `openapi-2.0.json`: https://spec.openapis.org/oas/2.0/schema/2017-08-27
- `openapi-3.0.json`: https://spec.openapis.org/oas/3.0/schema/2024-10-18
- `openapi-3.1.json`: https://spec.openapis.org/oas/3.1/schema/2025-02-13
- `asyncapi-2.6.0.json`: https://raw.githubusercontent.com/asyncapi/spec-json-schemas/master/schemas/2.6.0-without-$id.json
- `asyncapi-3.0.0.json`: https://raw.githubusercontent.com/asyncapi/spec-json-schemas/master/schemas/3.0.0-without-$id.json
